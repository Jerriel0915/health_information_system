## 2. 登录说明

所有接口（除登录外）需要在请求头中携带 Token：

```
Authorization: Bearer eyJhbGciOiJIUzUxMiJ9...
```

### 登录接口
```
POST /login
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123",
  "code": "",
  "uuid": ""
}
```

返回示例：
```json
{
  "code": 200,
  "msg": "操作成功",
  "token": "eyJhbGciOiJIUzUxMiJ9..."
}
```

---

## 3. 统一返回格式

所有接口统一返回格式：
```json
{
  "code": 200,       // 成功；其他值为失败
  "msg": "操作成功",  // 提示信息
  "data": {}         // 实际数据（对象或数组）
}
```

分页列表接口返回：
```json
{
  "code": 200,
  "msg": "查询成功",
  "total": 78,
  "rows": []
}
```

## 5. 统计接口通用说明

- 所有统计接口的返回数据结构统一为 `{name, value}` 或 `{name, value, ...}` 格式，方便前端 ECharts 直接渲染
- 参数带 `?` 表示可选，不传则查全部
- 统计接口不需要分页参数
- 无数据时返回空数组 `[]` 或 `0` 值，不会报错

## 7. 注意事项

1. 后端运行在 **8081** 端口，非默认 8080
2. 所有 `/detail/{id}` 接口（CRUD 的详情查询）路径中带了 `/detail/` 前缀，与若依默认的 `/{id}` 不同，前端 API 调用时注意路径拼接
3. 验证码已关闭，登录时 `code` 和 `uuid` 传空字符串即可
4. 后端启动命令：`java -jar ruoyi-admin.jar`（约6-8秒启动完成）
